Ngôn ngữ sử dụng : Python 3.
Công Nghệ sử dụng :
  • UI: Bootstrap, Template Django, Css
  • Core: Django
  • Database: Mysql
Role: USER
  • Page Register:
    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    + Khi submit form nhận được data, nếu đủ các trường dữ liệu thì save và điều hướng đến trang login
    + nếu thiếu data sẽ set form empty
    + nếu là method GET render view trang register
  • Page Login:
    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    + POST: - nếu có username password check user trong database => ĐÚNG select all các mặt hàng, lưu user vào lưu trữ phiên, render view home, if SAI render view login
    + POST: - nếu là đăng xuất xóa user khỏi lưu trữ phiên và điều hướng trang đến login
    + GET: -(Mặc định) render view trang login
  • Page Home:
    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    + POST: - nhập vào thanh tìm kiếm submit => select all sản phẩm lặp qua từng sản phẩm nếu "input" nằm trong tên sản phẩm, add vào 1 list mới, render list đã có ra view home
    + POST: - nhấn vào danh mục sản phẩm => select all theo danh muc
    + GET: - nhấn vào giỏ hàng => điều hướng đến trang giỏ hàng hiển thị những sản phẩm đã thêm từ trước
    + POST: - nhấn vào mua ngay => lấy sản phẩm theo id render view product
    + POST: - nhấn vào nút giỏ hàng => lấy sản phẩm theo id, thêm trạng thái vận chuyển, idUser đăng nhập(ở lưu trữ phiên) render view cart
    + GET: -(Mặc định) render view trang home
  • Page Product:
    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    + GET:(Mặc định) - render sản phẩm từ hành động nhấn vào nút mua ngay ở trên
    + POST: - chọn màu sắc nhấn mua => thêm trạng thái vận chuyển,idUser đăng nhập(ở lưu trữ phiên) render view cart
    + GET: -(Mặc định) render view trang product
  • Page Cart:
    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    nếu là sản phẩm mới thêm chưa có trạng thái vận chuyển thì có thể xóa,ngược lại thì không
    + POST: - submit => cập nhật trạng thái vận chuyển đơn hàng
    + POST: - nhấn vào trạng thái => hiển thị các sản phẩm của user đang login và trạng thái mặt hàng đó
    + GET: -(Mặc định) render view trang cart
Role: ADMIN

    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    + POST: - xóa => delete sản phẩm theo id
    + POST: - sữa => lấy sản phẩm theo id render view trang Add(page dùng cho cả thêm mới và sữa)
    + GET: -(Mặc định) render view trang admin

    FORM có 2 method GET && POST (mặc định là GET còn submit là POST)
    nếu là add thì không có sản phẩm còn update thì có sản phẩm cần update
    + POST: - submit => thêm mới ,update theo từng trường hợp
    + GET: -(Mặc định) render view trang Add
